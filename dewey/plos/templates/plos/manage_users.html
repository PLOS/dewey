Core Groups

Core groups determine what users get created on our deployed systems. If a user is a member of one of the core groups, that user will have an account created on all of our systems. Generally speaking, a user should be in only one core group.

Extra Groups

Extra groups generally exist to give their members extra permissions. The users added to an extra group must also exist in one of the core groups, or else the account will never be created.
